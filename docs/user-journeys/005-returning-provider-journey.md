# The Returning Provider — Status Update Journey

## Actor
A long-term provider updating clients on the status of services they are currently delivering.

## Goal
Communicate progress, manage expectations, and get paid.

## Steps
1. Logs in and navigates to active contracts.
2. Updates each contract with progress milestones and deliverables.
3. Receives client feedback and addresses revisions.
4. Marks contracts complete to trigger escrow release.

## Pain Points
- Clients who don't respond to updates or payment triggers.
- Scope changes mid-project without updated contracts.
- Lack of a structured way to log progress and communicate deliverables.

## Success Criteria
The provider updates all active contracts and successfully triggers at least one escrow release.
