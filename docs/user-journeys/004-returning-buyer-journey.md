# The Returning Buyer — Status Check Journey

## Actor
A long-term user checking on the status of ongoing services they previously purchased.

## Goal
Monitor progress, review deliverables, and manage active contracts.

## Steps
1. Logs in and navigates to their dashboard.
2. Reviews active contracts and delivery timelines.
3. Communicates with the provider on progress updates.
4. Releases escrow upon satisfactory completion.

## Pain Points
- Poor visibility into project progress — no real-time status updates.
- Missed deadlines or silent providers.
- Dispute resolution is unclear or slow.

## Success Criteria
The buyer reviews all active contracts and confirms or requests changes to each.
