# Architecture Decision Records (ADRs)

ADRs capture the *what*, *why*, and *context* of technical decisions so the team has a shared memory.

Each ADR is a single markdown file named `adr-NN.md` (e.g. `adr-001.md`).

## Naming convention

- Use zero-padded numbers: `adr-001.md`, `adr-002.md`, ...
- Title is the decision summary
- Follow the template in `../templates/adr-template.md`

## Statuses

- **proposed** — under discussion
- **accepted** — agreed and will be implemented
- **deprecated** — superseded by a newer decision
- **superseded** — replaced by another ADR
